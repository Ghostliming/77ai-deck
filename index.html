议题二 · 数据智能 · 应用实践

Insight · Question

一个常见的经营追问

经营管理者 这个季度收入看起来增长了，但利润没有同步变好——到底是哪些客户、项目、产品或部门在拉低毛利？

企企 ERP 里已有业务数据——难的是按管理视角呈现，并支撑经营分析与决策建议。

议题二 · 经营分析入口

议题二 R0。对老客户口播要点：不是「数进不了系统」，而是「有了数，怎么按管理问题看清楚、分析透、敢决策」。

Insight · Context

ERP 能承载业务数据
经营分析还缺什么？

维度企企 ERP 已提供经营分析中的缺口
业务归集合同、项目、收入、成本、费用等瓶颈不在录入——数据已在系统中
管理视角单据、标准报表、固定查询难按当期管理问题灵活组合呈现
口径对齐各模块各有统计口径跨模块对齐慢，「利润」含义易不一致
多源拼数对象内数据完整不同对象、外部参考数据需人工拼接
决策支持人读报表、人判断缺证据链，难快速归因到项目/成本项
经验复用分析师个人经验分析过程难沉淀为团队可复用资产
业务已归集 · 呈现与分析仍靠人

这页结构对齐议题一 CRM 缺口页，但前提不同：老客户知道 ERP 能管业务，缺口在呈现、分析、决策、沉淀。

财务/老板：固定报表答不了「这个季度为什么利润没跟上」。IT：拼数、口径、权限仍是痛点。

Insight · Value

构建可信 · 呈现直观 · 支撑根因

基于企企 ERP 业务数据与 AI 能力，构建面向经营问题的轻量级 BI 应用——按管理意图构建、查看与追问。

构建可信

呈现直观

图表、看板、筛选与下钻——高效、直观的数据呈现，敢看、敢用。

同类能力也可服务于客户健康度、续约分析等场景——同理可应用于 ARR / NDR / CLTV / 客户续约续费分析。

Insight · Practice

构建过程中的
实践认知

01Code 能力 · 交给 Agent 写

认知：LLM 代码能力已足够强，不必再堆配置模型

图表构建由 Agent 生成配置，而非构建配置模型；数据处理写代码完成；PaaS 平台脚本亦由 Agent 生产。

02边界与上下文 · 释放模型

认知：Harness 的价值在治理与连接，不在替模型套死流程

做好边界与上下文控制，充分释放模型能力——避免固化流程、把模型困在既定套路里。

03效果优先 · 质量飞轮

认知：软件 3.0 时代，效果取决于模型能力与工程构建

模型高频更新，工程需持续适配——效果观测与质量飞轮极其重要，上线不是终点。

04进入真实业务 · 而非新瓶装旧酒

坑：做成「更大号聊天框」、更复杂的固定报表，或低代码配置堆叠

解法：让 AI 进入真实业务过程——领域交互、数据切面、受控执行；关键动作人确认，能力与经验持续沉淀。

Code 生产 · 释放模型 · 质量飞轮 · 真实业务

口播重点：01 图表/数据/脚本都交给 Agent 写代码；02 边界上下文、勿固化困住模型；03 效果飞轮。04 收束产品形态，与 02 不重复——02 讲 Harness 原则，04 讲反模式。
